# Защита системы

В этой папке собраны механизмы, с помощью которых система защищает себя от прямой ответственности.

## Список защит

1. [Размывание ответственности через обобщение](./ResponsibilityDilutionByGeneralization.md)  
   Суть: вопрос о конкретном субъекте переводится в общее правило, и персональная ответственность исчезает из обсуждения.

2. [Подмена ответственности объяснением](./ResponsibilitySubstitutionByExplanation.md)  
   Суть: вопрос «кто отвечает» подменяется объяснением «почему так произошло», что создаёт иллюзию ответа без признания ответственности.

3. [Аннулирование по частному отличию](./AnnullingByPartialDifference.md)  
   Суть: при сущностном сходстве случаев система выпячивает локальное отличие и использует его, чтобы разрушить аналогию и уйти от признания паттерна.

4. [Редукция факта к интерпретации](./FactReductionToInterpretation.md)  
   Суть: указание на проверяемый факт переводится в «субъективную трактовку», чтобы заменить разбор причинности спором мнений.

5. [Деперсонализация ответственности через правило](./DepersonalizationOfResponsibilityThroughRule.md)  
   Суть: система ссылается на «процедуру» как на безличную неизбежность, чтобы скрыть автора правила и уйти от вопроса «кто принял решение».
