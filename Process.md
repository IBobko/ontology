# Что такое процесс

> Соседний слой: [Что такое событие](Event.md).

Этот документ фиксирует процесс как **последовательность связанных событий** — без подмены процесса единичным переходом.

## 1) Базовая установка

**Процесс — это не одно событие, а развёртка изменений во времени.**

Если событие фиксируется как минимальный переход:

```text
было -> стало иначе
```

то процесс удерживается как цепочка переходов:

```text
S0 -> S1 -> S2 -> ... -> Sn
```

Коротко:

```text
событие = переход
процесс = последовательность переходов
```

## 2) Минимальная внутренняя структура процесса

На рабочем уровне процесс содержит четыре обязательных слоя.

### 2.1 Последовательность событий

Процесс невозможен без множества событий, связанных по порядку:
- есть начальная конфигурация;
- есть промежуточные переходы;
- есть текущая или конечная конфигурация.

Без последовательности остаётся только единичное событие.

### 2.2 Длительность

Процесс всегда протяжён во времени:
- имеет начало;
- имеет ход;
- может иметь завершение или оставаться незавершённым.

Это принципиально отличает процесс от события как моментального перехода.

### 2.3 Связность переходов

События внутри процесса не просто стоят рядом, а образуют связку:
- по причинной линии;
- по функциональной линии;
- по организационной линии (правила, роли, обязательства).

Если связности нет, это набор несвязанных эпизодов, а не процесс.

### 2.4 Направление / вектор

Процесс удерживается как изменение с направлением:
- к цели;
- к распаду;
- к стабилизации;
- к неопределённости.

Вектор не равен оценке, но задаёт форму движения процесса.

## 3) Скелет процесса

```text
ПРОЦЕСС =
  [ПОСЛЕДОВАТЕЛЬНОСТЬ СОБЫТИЙ]
+ [ДЛИТЕЛЬНОСТЬ]
+ [СВЯЗНОСТЬ ПЕРЕХОДОВ]
+ [НАПРАВЛЕНИЕ]
```

Это не моральная трактовка, а минимальный структурный каркас.

## 4) Событие и процесс: строгое различие

```text
СОБЫТИЕ  = минимальный переход состояния
ПРОЦЕСС  = последовательность переходов
```

Примеры:
- «дверь закрылась» — событие;
- «дверь закрывается» — процесс.

- «война началась» — событие;
- «война идёт» — процесс.

- «машина тронулась» — событие;
- «машина едет» — процесс.

## 5) Почему процесс часто путают с событием

Язык часто сжимает протяжённое во времени в одно имя:
- «ремонт»;
- «поездка»;
- «переговоры»;
- «суд».

Формально это процессы, но в речи они могут подаваться как «одно событие».

Такое сжатие полезно для коммуникации, но ослабляет онтологическую точность.

## 6) Уровни фиксации

Удобно различать три последовательных слоя:

```text
изменение -> событие -> процесс
```

где:
- изменение — факт различия состояний;
- событие — момент фиксации изменения;
- процесс — последовательность связанных событий.

## 7) Фиксация для модели

1. Процесс всегда длится во времени.
2. Процесс не заменяет событие, а включает события как элементы.
3. Смешение процесса и события приводит к потере точности анализа.
4. Удержание процессной структуры снижает риск шаблонного сворачивания.
5. Для причинно валидной обработки нужно различать: изменение, событие, процесс.
